## Upgrade notes

v0.3.9 has no database schema changes and can be upgraded directly from v0.3.8.

## Changes

- Add two-finger pinch zoom to Mermaid charts on touch devices while retaining one-finger pan, wheel zoom, toolbar zoom, and reset controls.
- Improve wide Markdown tables on phones with an independent horizontal scroller, a sticky first column, readable column widths, and keyboard-accessible overflow regions.
- Render real Mermaid charts in single-report PDF exports after the report has been viewed. Export waits for pending chart uploads so clicking PDF immediately after opening a report no longer races the cache.
- Use the same browser-rendered Mermaid geometry for warmed reports in daily ZIP PDFs. The PDF path keeps JavaScript, external links, local file access, plugins, and image loading disabled.
- Add a bounded, user-scoped in-memory chart cache and a deny-by-default SVG re-serializer. PDF SVG accepts only URL-free geometry and text; Mermaid markers are flattened into ordinary polygons for wkhtmltopdf compatibility.

## Known limitations

- The in-memory PDF chart cache is warmed by opening a report in the browser. A cold daily ZIP export shows a clearly labelled Mermaid source fallback for reports that have not been viewed since the server started.
- Markdown downloads continue to preserve the complete Mermaid source.
- Mermaid `xychart-beta` does not support dual Y axes. Use stacked charts for price and RSI/KD series with different scales.
