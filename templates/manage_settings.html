新建令牌

入库（Dify 工作流末尾 HTTP 节点）

POST /api/reports · Header Authorization: Bearer <令牌> · JSON：
{"symbol","date","kind","subtype","title","body_md","run_id"} （kind 不填按 subtype 推断；同 标的+日期+大类+小文档 再入库即覆盖更新，run_id 只当批次标签）。
可选带结构化跟踪项："tracking":[{"itype":"assumption|tracking","content":"…","status":"pending","review_point":"…"}]

查 / 搜（Dify 复核历史假设与跟踪事项）· 全部走 query 作用域令牌

是否存在 / 是否过时（返回 has=true/false + 每篇信息）：
GET /api/reports?symbol=300750&date=today　或按天数：&since=2026-06-28
关键词查 / 搜（和网页一致，symbol 或 q 至少一个）：
GET /api/reports?symbol=&q=重组&kind=&subtype=&since=&until=&limit=20&with_body=1
探清单（某票有哪些日期/大类）：GET /api/reports/manifest?symbol=300750
报告组（一次生成收一组）：GET /api/runs?symbol=300750&date=
股票清单/补全（库里有哪些票）：GET /api/symbols?q=300&limit=50
结构化假设/跟踪项（重跑复核）：GET /api/tracking?symbol=300750&status=pending
取单篇正文：GET /api/report?uid=... → 含 body_md / body_html
